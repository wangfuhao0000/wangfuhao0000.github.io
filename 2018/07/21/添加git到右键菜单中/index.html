添加git到右键菜单中

2018.07.21 John Doe  热度 ℃

今天遇到一个问题，就是当时我安装visualstudio2017的时候直接就把git给安装了，但是后来发现在右键菜单中找不到，以往单独安装git的时候都会直接加到右键菜单的，但具体我忘了是在哪一步进行选择。但是没有的话，会很不方便，即使你可以在所有程序中找到，但不如右键来的舒服，于是到网上搜索了一下，找到了解决方法，只是最后成功解决时与他的有些出入，我搜索的是这篇博客右键菜单添加git选项。
接下来我就对他的进行总结，并针对我自己遇到的问题改一些地方。

  1. 使用win+r，输入regedit，进入到注册表编辑器。
  2. 找到 HKEY_CLASSES_ROOT > Directory > Background > shell
  3. shell右键点击 > 新建 > 项 > 输入名称 git_gui
  4. 点击新建的项git_gui > 右边框里有个选项，名称为默认 > 双击 > 输入数值数据 Git GUI
  5. 新建的项git_gui右键点击 > 新建字符串值 > 名称设置为 Icon > 数据设置为 git_gui.exe 的完整路径
  6. 新建的项git_gui右键点击 > 新建项 > 名称设置为 command > 数据设置为 git_gui.exe 的完整路径

这里是和博客中的一样的，但是要进行一些说明：

* 第 3条中的git_gui只是一个名字，它表示这个项的名字，和在右键菜单中展现出来的文字是无关的。
* 第 4条中的的数值数据Git GUI则是你要在右键菜单中展示出来的名字，它是什么，右键菜单中就会展示什么。
* 第 5条中的Icon则是设置一下这个菜单的图标位置 ，不设置也没关系，但是到时候在右键菜单展示时，它左边就没有图标，只有文字。
* 第 6条中的command的的数据为git_gui的完整路径。这个路径一开始我以为是程序所在路径，即在所有程序中找到它点击右键打开文件位置，这时候它会弹出文件夹，并指示一个路径。但事实证明，并不是这个路径，而是你点击程序的右键，查看它的属性，其中有个目标属性，这个才是此处要写的，即command的数值。例如我的程序所在地址为“C:\ProgramData\Microsoft\Windows\Start Menu\Programs\Git\Git GUI.exe”，但是此程序的“目标”属性确是"C:\Program Files\Git\cmd\git-gui.exe"。所以两者一定不要弄混，同理Git Bash的也是这样，这两个路径是不一样的，一定是程序的“目标”属性的值。

至于Git Bash的也和Git GUI的操作过程一样的，只是路径变为了Git Bash的位置。
